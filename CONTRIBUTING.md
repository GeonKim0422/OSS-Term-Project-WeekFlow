# WeekFlow Contributing Guidelines

Thank you for your interest in the WeekFlow project! Your contributions are vital to growing this open-source planner.

## ü§ù Ways to Contribute

There are many ways to contribute to WeekFlow, and not all of them involve writing code.

1. Bug Reports

If you find unexpected behavior or errors, please open a new issue.

Title: Keep it concise and descriptive (e.g., [Bug]: TimeBlock splitting error in auto-placement).

Content: Include Steps to Reproduce, Expected Behavior, Actual Behavior, and Environment details.

2. Feature Suggestions

New ideas are always welcome! Use the [Feat] or [Idea] prefix when creating an issue.

Context: Explain why this feature is needed and what problem it solves.

Proposed Implementation: Share any simple ideas you have for implementation.

3. Code Contributions

Create/Select Issue: Always ensure there is a corresponding issue for your work.

Fork & Branch: Fork the WeekFlow repository and create a new branch (e.g., feat/add-deadline-filter).

Code & Test: Write your code and run necessary tests.

Commit: Write clear and concise commit messages.

Recommended Format (Conventional Commits): type(scope): subject

feat: A new feature

fix: A bug fix

docs: Documentation only changes

chore: Build process or auxiliary tool changes

Submit Pull Request (PR):

Create a PR from your branch to the original repository's main branch.

Crucially, include the linked Issue number in the PR description (e.g., Closes #123).
